## 📋 功能说明

### 1. 💼 基金管理
**功能**: 添加或移除监控基金

**操作步骤**:
1. 点击"管理基金"按钮
2. 在输入框中输入基金代码（如: 001593）
3. 点击"添加"按钮
4. 查看已添加的基金列表
5. 需要移除时点击"移除"按钮

**说明**:
- 添加的基金会自动保存到配置文件
- 支持批量添加多个基金
- 移除后配置文件会同步更新

---

### 2. 📋 持仓获取
**功能**: 获取基金前十大重仓股信息

**操作步骤**:
1. 点击"获取持仓"按钮
2. 输入基金代码
3. 选择获取方式:
   - **从配置读取**: 使用本地缓存的持仓数据（速度快）
   - **联网更新**: 从网络重新获取最新持仓（数据新）

**显示信息**:
- 股票排名
- 股票代码
- 股票名称
- 持仓比例
- 总持仓比例

---

### 3. ✏️ 持仓编辑
**功能**: 手动编辑基金持仓比例

**操作步骤**:
1. 点击"编辑持仓"按钮
2. 输入基金代码，点击"加载持仓"
3. 修改持仓信息:
   - 修改股票代码
   - 修改股票名称
   - 修改持仓比例
   - 删除某个持仓
   - 添加新持仓
4. 点击"保存修改"

**使用场景**:
- 调整持仓比例进行估值测试
- 添加自定义股票
- 删除不需要的持仓

---

### 4. 💰 估值计算
**功能**: 实时计算基金估值涨跌

**操作步骤**:
1. 点击"计算估值"按钮
2. 点击"计算所有基金估值"
3. 等待计算完成
4. 查看估值结果

**显示信息**:
- 基金名称和代码
- 估算净值
- 估算涨跌幅（带颜色区分）
- 上次净值和日期
- 重仓股数量
- 持仓比例合计
- 估算时间

**颜色说明**:
- 🔴 红色: 上涨
- 🟢 绿色: 下跌

---

### 5. 📈 监控页面生成
**功能**: 生成包含K线和估值的综合监控页面

**操作步骤**:
1. 点击"生成页面"按钮
2. 确认生成
3. 等待生成完成（可能需要1-2分钟）
4. 提示成功后可选择立即查看

**生成内容**:
- 基金实时估值表格
- 股票指数K线图（日/周/月）
- 美观的页面布局

**输出文件**: `zs_fund_online.htm`

---

### 6. 👀 查看监控
**功能**: 打开已生成的监控页面

**操作步骤**:
1. 点击"查看页面"按钮
2. 在新标签页中打开监控页面

**前提条件**: 需要先生成监控页面

## 🔧 高级功能

### API接口
系统提供RESTful API，可用于集成:

```
GET  /api/config              - 获取配置
POST /api/config              - 保存配置
GET  /api/fund/holdings/{code} - 获取持仓
PUT  /api/fund/holdings/{code} - 更新持仓
GET  /api/fund/valuation/{code} - 单个估值
POST /api/fund/valuation/batch  - 批量估值
POST /api/generate/monitor      - 生成页面
GET  /api/monitor/view          - 查看页面
GET  /api/fund/list             - 基金列表
POST /api/fund/add              - 添加基金
DELETE /api/fund/remove/{code}  - 移除基金
```

## ⚠️ 注意事项

1. **端口占用**: 
   - 默认使用5000端口
   - 如果端口被占用，修改`web_server.py`中的端口号

2. **网络连接**:
   - 获取持仓和计算估值需要网络连接
   - 建议使用"从配置读取"提高速度

3. **数据安全**:
   - 所有数据保存在本地
   - 配置文件:`zs_fund_online.json`
   - 定期备份配置文件

4. **性能优化**:
   - 避免频繁联网更新
   - 建议使用缓存的持仓数据

## 🛠️ 故障排除

### 问题1: 无法启动服务器
**解决方案**:
```bash
# 检查Flask是否安装
pip install flask

# 检查端口是否被占用
netstat -ano | findstr :5000
```

### 问题2: 页面无法访问
**解决方案**:
- 确认服务器已启动
- 检查防火墙设置
- 尝试使用`http://127.0.0.1:5000`

### 问题3: 估值计算失败
**解决方案**:
- 检查网络连接
- 确认基金代码正确
- 查看控制台错误信息

### 问题4: 持仓获取失败
**解决方案**:
- 先尝试"联网更新"
- 检查基金代码是否有效
- 查看日志文件`FundValuationAPI.log`
