# 板块描述自动获取功能

## 功能概述

当用户在 `zs_all` 中添加了新的基金/指数，但在 `zs_descriptions` 中没有对应描述时，系统会自动联网获取描述。获取优先级为：

1. **预定义描述** - 内置的63个常用板块详细描述
2. **联网获取** - 从东方财富API获取实时信息
3. **默认生成** - 基于板块名称关键词自动生成

## 自动触发场景

### 1. 服务器启动时
服务器启动时会自动检查配置，发现缺失描述的板块时会自动获取：
```
[描述初始化] 检测到 2/63 个板块缺少描述，自动获取中...
[描述自动获取] 新板块ABC(1.999999) - 使用预定义描述
[描述自动获取] 新能源ETF(2.888888) - 正在联网获取...
[描述自动获取] 完成！成功: 2, 失败: 0
```

### 2. 用户添加新板块后
通过管理后台保存配置时，系统会在后台自动获取新增板块的描述：
```python
# save_config 函数中自动触发
threading.Thread(target=fetch_descriptions_async, daemon=True).start()
```

### 3. API手动触发
提供API端点供手动触发获取：

#### 刷新所有缺失描述
```http
POST /api/index/descriptions/refresh
```

响应示例：
```json
{
  "success": true,
  "message": "描述刷新成功，新增 2 个板块描述",
  "data": {
    "count": 65,
    "missing_fetched": 2,
    "descriptions": {...}
  }
}
```

#### 获取单个板块描述（自动获取）
```http
GET /api/index/description/{index_code}
```
如果描述不存在，会自动获取并返回。

#### 强制重新获取单个板块描述
```http
POST /api/index/description/{index_code}/fetch
```

响应示例：
```json
{
  "success": true,
  "message": "描述获取成功",
  "data": {
    "code": "1.000698",
    "name": "科创100",
    "description": "科创100指数是科创板中小市值成长股的代表指数...",
    "source": "online"
  }
}
```

## 数据来源

### 1. 预定义描述（离线）
内置63个常用板块的详细描述，包括：
- 宽基指数（上证指数、沪深300、科创50/100等）
- 行业板块（半导体、新能源、医药、消费等）
- 海外市场（纳斯达克、日经、恒生等）

### 2. 东方财富API（在线）
```python
# 搜索API
EASTMONEY_QUOTE_API = "https://searchapi.eastmoney.com/api/suggest/get"

# 详情API  
EASTMONEY_KLINE_API = "https://push2.eastmoney.com/api/qt/stock/get"
```

### 3. 智能默认生成
基于板块名称关键词匹配生成：
```python
descriptions_map = {
    '科创': '科创板相关指数，聚焦科技创新企业...',
    '新能源': '覆盖新能源汽车、光伏、风电等清洁能源领域...',
    '半导体': '聚焦半导体产业链，涵盖芯片设计、制造、封测等环节...',
    # ... 更多关键词
}
```

## 配置说明

### 环境变量
```bash
# 禁用联网获取（仅使用预定义和默认描述）
export ENABLE_INDEX_DESC_FETCH=false

# 启用联网获取（默认）
export ENABLE_INDEX_DESC_FETCH=true
```

### 配置文件结构
```json
{
  "zs_all": {
    "1.000698": ["科创100", "019861,019867"],
    "1.999999": ["新板块", "000001"]
  },
  "zs_descriptions": {
    "1.000698": "科创100指数是科创板中小市值成长股的代表指数...",
    "1.999999": "新板块是反映相关板块整体表现的重要指数。"
  }
}
```

## 使用示例

### 场景1：添加新板块
1. 用户在 `zs_all` 中添加新板块 `{"2.999999": ["我的板块", "000001"]}`
2. 保存配置时，系统自动在后台获取描述
3. 刷新页面，新板块显示自动生成的描述

### 场景2：手动刷新描述
```bash
# 调用API刷新所有缺失描述
curl -X POST http://localhost:5000/api/index/descriptions/refresh
```

### 场景3：手动获取单个板块
```bash
# 获取单个板块描述（不存在时自动获取）
curl http://localhost:5000/api/index/description/2.999999

# 强制重新获取
curl -X POST http://localhost:5000/api/index/description/2.999999/fetch
```

## 注意事项

1. **网络依赖** - 联网获取需要访问东方财富API，请确保网络畅通
2. **请求频率** - 自动获取时会添加0.3秒延迟，避免请求过快被封
3. **失败回退** - 联网失败时会自动使用预定义或默认描述
4. **异步处理** - 配置保存后的自动获取是异步执行的，不会阻塞响应
