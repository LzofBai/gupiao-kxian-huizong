# 基金实时估值功能

## 功能概述

**功能名称**: 基金实时估值（基于持仓股票估算）  
**技能类型**: 数据分析计算  
**版本**: 1.0.0  
**创建日期**: 2026-02-08  
**依赖模块**: `FundValuationAPI.py`

本功能通过获取基金前十大重仓股的实时行情数据，结合持仓比例计算基金的实时估值，弥补官方网站关闭实时估值功能后的空缺。

---

## 功能特点

### ✅ 核心优势
- 🎯 **精准估算**: 基于前十大重仓股实时行情计算
- 📊 **持仓透明**: 显示每只重仓股的涨跌贡献度
- ⚡ **实时更新**: 获取股票实时行情数据
- 🔄 **批量处理**: 支持同时计算多个基金估值
- 📝 **日志记录**: 完整的操作日志便于追踪

### 📈 计算原理

```
估算涨跌幅 = Σ(重仓股涨跌幅 × 持仓比例)
估算净值 = 上次净值 × (1 + 估算涨跌幅)
```

**示例计算**:
- 股票A: 涨跌 +2%, 持仓 10% → 贡献 +0.20%
- 股票B: 涨跌 -1%, 持仓 8%  → 贡献 -0.08%
- 股票C: 涨跌 +3%, 持仓 7%  → 贡献 +0.21%
- **总估算涨跌**: +0.33%

## 数据来源

### API接口

| 数据类型 | 数据源 | 接口地址 |
|---------|--------|---------|
| 基金基本信息 | 天天基金网 | `http://fundgz.1234567.com.cn/js/{code}.js` |
| 基金持仓信息 | 东方财富网 | `http://fundf10.eastmoney.com/FundArchivesDatas.aspx` |
| 股票实时行情 | 东方财富网 | `http://push2.eastmoney.com/api/qt/stock/get` |

### 数据字段说明

**基金信息**:
- 基金代码、基金名称
- 上次净值、净值日期
- 估值、估值时间、日涨跌幅

**持仓信息**:
- 股票代码、股票名称
- 持仓比例（%）

**股票行情**:
- 最新价、涨跌幅、涨跌额
- 昨收价

## 实战案例

### 案例1: 自选基金监控脚本

```python
from FundValuationAPI import FundValuationAPI
from datetime import datetime

# 我的自选基金
my_funds = {
    '001593': '创业板',
    '001549': '上证50',
    '012842': '军工',
    '007531': '券商'
}

api = FundValuationAPI()

print(f"\n{'='*60}")
print(f"我的基金监控 - {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
print(f"{'='*60}\n")

for code, name in my_funds.items():
    result = api.calculate_fund_valuation(code)
    if result:
        change = result['估算涨跌幅']
        emoji = '📈' if change > 0 else ('📉' if change < 0 else '➡️')
        print(f"{emoji} {name:8} {result['估算净值']:.4f} "
              f"({change:+.2f}%)  [{result['重仓股数量']}只重仓]")
```

### 案例2: 定时监控并发送通知

```python
import schedule
import time
from FundValuationAPI import FundValuationAPI

api = FundValuationAPI()

def check_funds():
    """定时检查基金涨跌"""
    fund_codes = ['001593', '001549', '012842']
    results = api.batch_calculate_valuations(fund_codes)
    
    for code, val in results.items():
        change = val['估算涨跌幅']
        
        # 涨跌超过1%发送提醒
        if abs(change) > 1.0:
            print(f"⚠️ 提醒: {val['基金名称']} "
                  f"涨跌达到 {change:+.2f}%")

# 交易日每5分钟检查一次
schedule.every(5).minutes.do(check_funds)

while True:
    schedule.run_pending()
    time.sleep(60)
```

### 案例3: 生成每日报表

```python
from FundValuationAPI import FundValuationAPI
from datetime import datetime
import json

api = FundValuationAPI()

fund_codes = ['001593', '001549', '004742', '011615']
results = api.batch_calculate_valuations(fund_codes)

# 生成JSON报表
report = {
    '报表时间': datetime.now().strftime('%Y-%m-%d %H:%M:%S'),
    '基金数量': len(results),
    '基金详情': []
}

for code, val in results.items():
    report['基金详情'].append({
        '代码': code,
        '名称': val['基金名称'],
        '估值': val['估算净值'],
        '涨跌': val['估算涨跌幅']
    })

# 保存报表
filename = f"fund_report_{datetime.now().strftime('%Y%m%d_%H%M')}.json"
with open(filename, 'w', encoding='utf-8') as f:
    json.dump(report, f, ensure_ascii=False, indent=2)

print(f"报表已保存: {filename}")
```

## 注意事项

### ⚠️ 重要说明

1. **估值准确性**
   - 仅基于前十大重仓股计算，不包含全部持仓
   - 持仓数据可能存在滞后（通常为季度更新）
   - 不考虑其他资产（债券、现金等）
   - **仅供参考，不作为投资依据**

2. **数据时效性**
   - 股票行情为实时数据
   - 持仓信息为最近一期公开数据
   - 交易日 9:30-15:00 期间估值最准确

3. **使用限制**
   - 避免过度频繁请求，建议间隔≥5秒
   - 遵守数据源网站的使用条款
   - 仅供个人学习研究使用

4. **错误处理**
   - 网络异常会返回None
   - 部分基金可能无法获取持仓数据
   - 股票停牌会影响估值准确性

## 故障排除

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|-----|---------|---------|
| 获取不到持仓数据 | 基金代码错误 | 检查基金代码是否正确 |
| 估值为空 | 网络连接失败 | 检查网络连接 |
| 持仓股票很少 | 解析失败 | 查看日志文件排查 |
| 估值不准确 | 持仓数据过旧 | 等待基金季报更新 |
