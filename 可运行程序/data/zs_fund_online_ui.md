# zs_fund_online_ui.json 文件结构说明

该文件是基金/指数监控系统的**数据配置文件**，用于定义需要监控的指数、板块及其关联基金代码。
该文件能记录监测的指数、板块的“一句话”简要描述，用于在主页面加载时将该描述显示在相关基金的标签处，用户可根据需求自由编辑，运行时由程序自行加载。
---

## 文件结构概览

```json
{
  "zs_all": { ... },
  "type_all": [ ... ],
  "last_migration": "..."
  "zs_descriptions": "..."
}
```

---

## 字段详解

### 1. `zs_all` - 指数/板块配置

- **类型**：对象（字典）
- **作用**：定义需要监控的指数/板块及其关联基金

#### 键（Key）
指数代码，格式说明：
| 前缀 | 含义 | 示例 |
|------|------|------|
| `1.` | 上证指数 | `1.000300`（沪深300） |
| `0.` | 深证指数 | `0.399006`（创业板指） |
| `90.BK` | 同花顺板块 | `90.BK0701`（中证500） |
| `2.` | 中证指数 | `2.931186`（中证科技） |
| `124.` | 恒生指数 | `124.HSTECH`（恒生科技） |
| `100.` | 海外指数 | `100.NDX`（纳斯达克） |
| `102.` | 商品期货 | `102.CL00Y`（NYMEX原油） |

#### 值（Value）
数组格式：
- **第1项**：指数/板块中文名称（如"沪深300"、"半导体"）
- **第2项及以后**：关联的基金代码，多个基金用逗号分隔

#### 示例
```json
"1.000300": ["沪深300", "007045,005918"],
"90.BK1036": ["半导体", "020629,014194"],
"124.HSTECH": ["恒生科技", "012349", "015283"]
```

---

### 2. `type_all` - 监控周期

- **类型**：字符串数组
- **作用**：定义支持的K线周期类型
- **值说明**：
  - `D`：日线（Day）
  - `W`：周线（Week）
  - `M`：月线（Month）

```json
"type_all": ["D", "W", "M"]
```

---

---

### 3. `last_migration` - 数据迁移时间戳

- **类型**：字符串
- **作用**：记录最后数据迁移/更新时间
- **格式**：`YYYY-MM-DD HH:MM:SS`

```json
"last_migration": "2026-02-09 12:03:12"
```

---

### 4. `"zs_descriptions" - 指数、板块描述

- **类型**：字符串
- **作用**：记录监测的指数、板块的“一句话”简要描述
- **格式**："指数代码”：“一句话描述”

```示例
 "1.000001": "上证指数是上海证券交易所编制的综合股价指数，反映沪市整体走势，是中国A股市场最重要的基准指数之一。"
```

## 数据覆盖范围

| 分类 | 包含内容 |
|------|----------|
| **宽基指数** | 沪深300、创业板指、中证500、上证50、科创50、北证50等 |
| **行业板块** | 半导体、医药、医疗、银行、券商、军工、传媒等 |
| **主题投资** | 人工智能、大数据、新能源、光伏、稀土、智能汽车等 |
| **海外指数** | 纳斯达克、日经225、越南胡志明、印度孟买SENSEX等 |
| **商品** | NYMEX原油、黄金 |
| **港股** | 恒生科技、恒生医疗保健 |

---

## 使用说明

1. **添加新指数**：在 `zs_all` 中添加新的键值对，键为指数代码，值为 `[名称, 基金代码]` 数组
2. **修改关联基金**：编辑对应指数数组中的基金代码，多个代码用逗号分隔
3. **调整监控周期**：修改 `type_all` 数组，可增减周期类型
4. **更新迁移时间**：数据变更后建议更新 `last_migration` 字段
